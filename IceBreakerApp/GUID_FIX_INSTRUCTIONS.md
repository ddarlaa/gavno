# Инструкции по исправлению проблем с GUID в проекте

## Проблема
В вашем проекте обнаружены проблемы с форматированием GUID:
1. Экранированные обратные слеши в GUID (\\\\)
2. Неправильное форматирование GUID без дефисов
3. Дублированные дефисы в GUID

## Решение

### 1. Замена проблемного файла SeedInitialData.cs
```bash
# Скопируйте исправленный файл в папку Migrations
copy "SeedInitialData_Fixed.cs" "C:\Users\dpugi\RiderProjects\IceBreakerApp\Migrations\SeedInitialData.cs"
```

### 2. Автоматическая проверка и исправление всех файлов проекта
```powershell
# Запустите PowerShell скрипт для проверки и исправления всех GUID
.\Fix-GuidInProject.ps1
```

### 3. Ручная проверка проекта
```bash
# Запустите batch файл для быстрой проверки
.\check_guids.bat
```

## Что было исправлено в SeedInitialData.cs

1. **Правильное форматирование GUID**: Все GUID теперь имеют стандартный формат `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`
2. **Использование Guid.Parse()**: Для гарантии корректного парсинга GUID
3. **Удаление экранированных символов**: Убраны все лишние обратные слеши
4. **Стандартизированные значения**: Все GUID используют понятные паттерны для тестирования

## Примеры исправлений

### Было:
```
Guid.Parse("deadbeef\\dead\\beef\\dead\\beefdeadbeef")
```

### Стало:
```  
Guid.Parse("deadbeef-dead-beef-dead-beefdeadbeef")
```

### Было:
```
"11111111222222223333333344444444"
```

### Стало:
```
"11111111-2222-2222-3333-333333333333"
```

## После применения исправлений

1. Пересоберите проект: `dotnet build`
2. Запустите миграции: `dotnet run`
3. Ошибка "Guid should contain 32 digits with 4 dashes" должна исчезнуть

## Дополнительные рекомендации

1. **Валидация GUID**: Используйте `Guid.TryParse()` для обработки пользовательского ввода
2. **Константы**: Рассмотрите возможность вынесения GUID в отдельный класс констант
3. **Логирование**: Добавьте логирование при создании GUID для отладки
4. **Тестирование**: Создайте юнит-тесты для проверки корректности GUID
